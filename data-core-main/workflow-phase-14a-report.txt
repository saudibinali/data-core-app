1. CURRENT AUTH / SUPER ADMIN MODEL REVIEW
================================================================================

Auth source:
  Self-hosted JWT. No external auth provider.
  POST /auth/login accepts employeeNumber + bcrypt password → signs JWT.
  Token stored in localStorage as "ops_access_token".
  Token payload: { userId, workspaceId, role }.

Users table (before P14-A):
  Table: users
  Role field: role TEXT — values: member, admin, manager, super_admin
  Status field: status TEXT — values: active, inactive
  Platform user detection: role = "super_admin" AND workspaceId IS NULL
  No separate platform_users table.

Super admin detection:
  requireSuperAdmin middleware: req.userRole !== "super_admin" → 403.
  No flag, no separate table — purely role-based.
  workspaceId IS NULL for all super_admin users (no workspace assignment).

SuperAdminRoute (frontend):
  Component guard: if role !== "super_admin" → redirect to /sign-in.
  All /super-admin/* routes protected by this guard.

Activity logs:
  activityLogsTable — userId (actor), action (string), metadata (JSON text),
  workspaceId (null for platform-level actions), createdAt.
  Pattern: insert BEFORE returning response (write-then-respond).

Password management:
  bcrypt (cost 12) hash stored in users.passwordHash.
  No external identity provider. No SSO. No MFA.

Migration impact:
  3 new columns added to usersTable: platformRoleCode, isRootOwner, isProtected.
  1 new column: lastLoginAt (timestamp, nullable).
  All additive — zero impact on existing records.
  Existing super_admin account: platformRoleCode = NULL, isRootOwner = false.
  Backward-compat root detection: role = "super_admin" AND workspaceId IS NULL
  AND platformRoleCode IS NULL → treated as Root Platform Owner.

Limitations:
  - requireSuperAdmin used for all platform user routes (full permission matrix
    deferred to P14-B).
  - No granular per-role platform access control yet.
  - lastLoginAt populated at login time — currently NULL for all existing users
    (login route update deferred to a separate micro-phase).

2. PLATFORM USER MODEL
================================================================================

Decision: NO new table created.
  Existing usersTable extended with 4 new columns.
  Platform users = usersTable WHERE workspaceId IS NULL.
  Preserves backward compatibility. No duplicate records. No FK inconsistency.

New columns added to usersTable:

  platformRoleCode  TEXT NULLABLE
    Sub-role within platform administration.
    NULL for the legacy Root Platform Owner (backward compat).
    Set to one of 7 assignable codes for newly created platform users.

  isRootOwner       BOOLEAN NOT NULL DEFAULT FALSE
    Explicit root flag. For future use (post P14-B migration).
    Currently unused at insert — root is derived from platformRoleCode IS NULL.

  isProtected       BOOLEAN NOT NULL DEFAULT FALSE
    Explicit protection flag. For future use (post P14-B migration).
    Currently derived at query time from isRootPlatformOwner().

  lastLoginAt       TIMESTAMP WITH TIMEZONE NULLABLE
    Last successful authentication. NULL = never signed in.
    Login route update (setting this on POST /auth/login) deferred.

PlatformUserStatus (4 values):
  invited   — created, pending first sign-in
  active    — active and can sign in
  disabled  — blocked, sign-in rejected by requireAuth middleware
  locked    — locked, sign-in rejected by requireAuth middleware

InitialPlatformRoleCode (8 values):
  root_platform_owner  — legacy root, NOT assignable from UI/API
  platform_admin       — full platform admin (assignable)
  support_admin        — support operations (assignable)
  workspace_support    — workspace read access (assignable)
  sales_admin          — subscription/plan read (assignable)
  finance_admin        — billing metadata read (assignable)
  auditor              — read-only audit access (assignable)
  read_only_operator   — minimal read-only access (assignable)

Root detection function (isRootPlatformOwner):
  Returns true when:
  (a) user.isRootOwner === true (explicit flag, future), OR
  (b) user.role === "super_admin" AND workspaceId IS NULL
      AND platformRoleCode IS NULL (backward compat for seeded admin).

requireAuth middleware update:
  Now also rejects users with status "disabled" or "locked" (not just "inactive").
  Zero impact on existing workspace users (their status is "active" or "inactive").

4. BACKEND API SURFACE
================================================================================

All routes: /api (prefix) — requireAuth + requireSuperAdmin

Method  Path                                   Description
──────  ─────────────────────────────────────  ─────────────────────────────────
GET     /platform/users                        List all platform users
GET     /platform/users/:userId                Get single platform user by ID
POST    /platform/users                        Create non-root platform user
PATCH   /platform/users/:userId/status         Change platform user status

Route file: artifacts/api-server/src/routes/platform-users.ts
Registered in: artifacts/api-server/src/routes/index.ts

GET /platform/users:
  - Filters: usersTable WHERE workspaceId IS NULL
  - Returns: { users: PlatformUserProfile[], total: number }
  - Read-only. No pagination in P14-A.
  - Audit: req.log.info (no DB write for reads)

GET /platform/users/:userId:
  - Returns: { user: PlatformUserProfile }
  - 404 if user not found or has a workspaceId (workspace user)
  - Audit: req.log.info

POST /platform/users:
  - Payload: { email, displayName, roleCode }
  - Validations:
    - email must be valid (regex)
    - displayName must be ≥ 2 chars
    - roleCode must be one of 7 assignable codes (not root_platform_owner)
    - actor must have higher privilege than the target role
    - email must be unique across all users
  - Defaults: status = "invited", isRootOwner = false, isProtected = false,
    workspaceId = null, role = "super_admin"
  - Audit: writes to activity_logs with action "platform_user_created"
  - Blocked root creation: writes "platform_user_create_blocked" + returns 403

PATCH /platform/users/:userId/status:
  - Payload: { nextStatus, reason, confirmation }
  - Validations:
    - nextStatus must be one of: active, disabled, locked
    - reason.trim().length ≥ 10
    - confirmation must be true
    - target must NOT be root/protected
    - actor must have higher privilege than target
    - actor cannot change their own status
  - Audit: writes "platform_user_status_changed" on success
           writes "protected_root_action_blocked" if root protection triggered
           writes "platform_user_status_change_blocked" otherwise

Safety — NOT implemented (no endpoint, no code):
  ❌ DELETE /platform/users/:userId — not implemented
  ❌ PATCH /platform/users/:userId/password — not implemented
  ❌ PATCH /platform/users/:userId/email — not implemented
  ❌ PATCH /platform/users/:userId/role — not implemented (P14-B)
  ❌ POST /platform/users/:userId/invite — not implemented
