عنوان المهمة:
WORKFLOW PHASE 13-B — Workspace Lifecycle Management & Controlled State Transitions

المهمة الحالية:
تحويل Tenant Registry من read-only visibility إلى controlled lifecycle administration.

الهدف:
تمكين السوبر أدمن من تنفيذ انتقالات حالة Workspace بشكل آمن ومراقب:
- activate
- suspend
- restore
- lock
- archive

بطريقة:
controlled, audited, deterministic, non-destructive lifecycle management

2 — LIFECYCLE STATE MACHINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

صمّم state machine واضحة:

WorkspaceLifecycleState:
- pending_activation
- active
- suspended
- locked
- archived

Compatibility mapping:
- workspaces.status = active    → active
- workspaces.status = suspended → suspended
- workspaces.status = disabled  → archived أو locked حسب الموجود

Allowed transitions:

pending_activation → active
active → suspended
active → locked
active → archived
suspended → active
suspended → locked
suspended → archived
locked → active
locked → suspended
locked → archived
archived → active فقط إذا restore مسموح
archived → suspended اختياري إذا منطقي

ممنوع:
- transition لنفس الحالة بدون سبب
- delete
- hard destroy
- bypass audit
- unknown → destructive state

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3 — LIFECYCLE ACTION MODEL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

صمّم:

WorkspaceLifecycleAction:
- activate
- suspend
- restore
- lock
- archive

لكل action:
- label
- description
- allowedFrom
- targetState
- severity
- requiresReason: true
- requiresConfirmation: true
- isDestructive: false
- auditEventType

مهم:
archive هنا soft/archive status فقط، ليس حذف بيانات.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
4 — REASON / CONFIRMATION CONTRACT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

كل lifecycle action لازم يطلب:
- reason
- confirmation checkbox أو typed confirmation
- optional internalNote
- actorId من session
- timestamp
- previousState
- targetState

Validation:
- reason required
- reason min length مثلاً 10
- target transition must be allowed
- actor must be super-admin
- tenant must exist
- no self-contradicting action

  "action": "suspend" | "activate" | "restore" | "lock" | "archive",
  "reason": "...",
  "internalNote": "... optional",
  "confirmation": true
}

- requireAuth

- load workspace
- derive current lifecycle state
- validate action allowed
