WORKFLOW PHASE 15-A — Commercial Accounts & Billing Contacts

المهمة الحالية:
تأسيس الحساب التجاري وجهات تواصل الفوترة لكل مستأجر/Workspace داخل Platform Administration.

الهدف:
تمكين موظفي المنصة المصرح لهم من:
- إنشاء أو تحديث Commercial Account للمستأجر
- تحديد Billing Contacts
- تحديد Account Manager
- تحديد Finance Contact
- تحديد Contract Owner
- حفظ Commercial Notes
- ربط هذه البيانات بالمستأجر الحالي
- عرضها داخل Tenant Admin Console من جهة المنصة

بدون:
دفع إلكتروني، فواتير رسمية، عقود كاملة، PO، تحصيل، أو ظهور للعميل داخل Workspace.

commercial_accounts — الحسابات التجارية

Fields:
- id — المعرّف
- tenantId / workspaceId — المستأجر أو مساحة العمل
- commercialAccountName — اسم الحساب التجاري
- legalEntityName — الاسم القانوني للجهة
- accountManagerUserId — مدير الحساب من مستخدمي المنصة
- financeOwnerUserId — مسؤول المالية الداخلي

- updatedBy — آخر من عدّل السجل
- createdAt
- updatedAt

- draft — مسودة
- active — نشط
- under_review — تحت المراجعة
- inactive — غير نشط

- one commercial account per tenant/workspace في P15-A
- لا يوجد invoice
- لا يوجد payment
- لا يوجد contract lifecycle كامل
- لا يوجد tax calculation
- companyTaxNumberPlaceholder مجرد حقل نصي اختياري

أضف APIs تحت Platform Administration:

GET /platform/tenants/:tenantId/commercial-account
Permission:
- commercial.accounts.read

Returns:
- commercial account
- contacts فقط إذا user عنده commercial.contacts.read
- إذا ما فيه account يرجع null أو draft-ready response

PUT /platform/tenants/:tenantId/commercial-account
Permission:
- commercial.accounts.update

Creates or updates one commercial account for tenant.

Payload:
- commercialAccountName
- legalEntityName
- accountManagerUserId
- financeOwnerUserId
- contractOwnerName
- contractOwnerEmail
- billingEmail
- billingPhone
- companyTaxNumberPlaceholder
- commercialNotes
- status

Rules:
- validate tenant exists
- validate accountManagerUserId/financeOwnerUserId are platform users if provided
- validate emails
- no payment fields
- no card fields
- no invoice fields
- audit:
  - commercial_account_created
  - commercial_account_updated

GET /platform/tenants/:tenantId/commercial-contacts
Permission:
- commercial.contacts.read

POST /platform/tenants/:tenantId/commercial-contacts
Permission:
- commercial.contacts.update

Creates billing contact.

PATCH /platform/tenants/:tenantId/commercial-contacts/:contactId
Permission:
- commercial.contacts.update

Updates billing contact.

PATCH /platform/tenants/:tenantId/commercial-contacts/:contactId/primary
Permission:
- commercial.contacts.update

Sets primary contact, ensures only one primary.

مهم:
لا تضف delete contact في P15-A.
إذا احتجت إزالة، استخدم status لاحقًا، لكن الأفضل تأجيل.
لا ترسل email.
