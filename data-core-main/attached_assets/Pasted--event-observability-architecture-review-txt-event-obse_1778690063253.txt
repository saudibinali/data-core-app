ثم نفّذ فقط:
PHASE 2 — Diagnostics APIs & Operational Visibility

بدون أي external telemetry systems أو observability overengineering.

عنوان المهمة:
EVENT DIAGNOSTICS PHASE 2 — Operational Visibility APIs

المهمة الحالية:
بناء طبقة Diagnostics تشغيلية حقيقية فوق الـ Event Platform الحالية.

الهدف:
تمكين:
- event inspection
- workflow tracing
- listener diagnostics
- operational debugging
- admin visibility

داخل النظام نفسه عبر APIs structured وقابلة للبناء عليها لاحقًا في Admin UI.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1 — EVENT DETAIL API
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

أنشئ endpoint:

GET /api/events/:id

المقصود:
workspace_event_logs.id

المطلوب:
إرجاع:
- event الأساسية
- payload
- result.listeners
- status
- error
- createdAt
- processedAt

مع enrichments التالية:

A) linked activity_logs
عبر:
activity_logs.bus_event_id
↔ payload._busEventId

B) linked notifications
عبر:
notifications.bus_event_id

C) linked workflow_executions
عبر:
workflow_executions.trigger_event_log_id

D) derived diagnostics
مثل:
- totalListenerDuration
- failedListenersCount
- notificationsCount
- workflowCount

المطلوب:
- workspace isolation
- proper auth
- no N+1 disasters
- pagination فقط إذا لزم للـ notifications/activity arrays

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
2 — WORKFLOW EXECUTIONS LIST API
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

أنشئ endpoint:

GET /api/workflows/executions

المطلوب:
filters:
- status
- workflowId
- workspaceId
- triggeredBy
- date range

إرجاع:
- execution الأساسية
- workflow name/key
- trigger event name
- triggerEventLogId
- startedAt
- completedAt
- computed duration
- currentStepIndex
- status

المطلوب:
- workspace isolation
- indexed queries
- no deep payloads بالقائمة

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3 — WORKFLOW EXECUTION DETAIL API
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

أنشئ endpoint:

GET /api/workflows/executions/:id

المطلوب:
- execution الأساسية
- triggering event
- linked workspace_event_log
- workflow definition metadata
- ordered execution steps timeline

لكل step:
- step name/type
- status
- startedAt
- completedAt
- computed duration
- input/output summary
- error summary

المطلوب:
- clean timeline structure
- no giant raw payloads
- redact sensitive fields إذا لزم
