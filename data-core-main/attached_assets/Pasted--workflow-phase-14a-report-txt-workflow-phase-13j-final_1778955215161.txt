عنوان المهمة:
WORKFLOW PHASE 14-B — Platform Roles & Permission Matrix

المهمة الحالية:
بناء مصفوفة صلاحيات داخلية لمستخدمي إدارة المنصة، وربطها تدريجيًا بالـ APIs والواجهة بدون كسر requireSuperAdmin الحالي.

الهدف:
تحويل أدوار Platform Users من مجرد roleCode إلى صلاحيات فعلية قابلة للتحكم مثل:
- platform.users.read — قراءة مستخدمي المنصة
- platform.users.create — إنشاء مستخدمي المنصة
- platform.users.status.update — تغيير حالة مستخدم منصة
- platform.users.role.update — تغيير دور مستخدم منصة
- tenants.read — قراءة المستأجرين
- tenants.lifecycle.update — تحديث دورة حياة المستأجر
- subscriptions.read — قراءة الاشتراكات
- subscriptions.update — تحديث بيانات الاشتراك
- entitlements.read — قراءة الصلاحيات والميزات
- entitlements.override.update — تحديث الاستثناءات
- usage.read — قراءة الاستخدام
- renewal.read — قراءة التجديد
- health.read — قراءة صحة المستأجر
- evaluation.read — قراءة التقييم
- audit.read — قراءة السجلات
- platform.activity.read — قراءة نشاط المنصة

بطريقة:
permission matrix foundation
مع الحفاظ على حماية Root
وبدون بناء نظام IAM كامل أو SSO/MFA

3 — PERMISSION MIDDLEWARE

أضف middleware:

requirePlatformPermission(permissionCode)

Behavior:
- requireAuth أولًا
- يقرأ user من req
- يستخرج platformRoleCode أو legacy root
- إذا user ليس platform user → 403
- إذا لا يملك permission → 403
- يسجل log/audit مناسب:
  - platform_permission_denied
- إذا يملك permission → next()

مهم:
لا تكسر requireSuperAdmin.
خليه موجود للتوافق.
لكن ابدأ باستخدام requirePlatformPermission في routes الجديدة أو المعدلة.

أضف helper:
requireAnyPlatformPermission([...])

اختياري إذا احتجته.
