LEAVE DOMAIN VALIDATION SUITE — Lifecycle, Concurrency & Integrity Testing

- lifecycle transitions صحيحة

- concurrency handling يعمل
- events لا تتكرر

لا تعدّل architecture.
فقط اختبارات + fixes صغيرة إذا ظهرت bugs حقيقية.

1 — LIFECYCLE TEST MATRIX
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

اختبر السيناريوهات التالية فعليًا عبر API requests:

A) Approval Flow
pending_approval → approved

تحقق من:
- leave_requests.status
- leave_approval_steps.status
- pending decrement
- used increment
- leave.approved emitted مرة واحدة
- notification واحدة فقط
- activity log واحد فقط

B) Rejection Flow
pending_approval → rejected

تحقق:
- pending restored
- used unchanged
- rejection event emitted مرة واحدة
- approval step updated correctly

C) Withdraw Flow
pending/pending_approval → withdrawn

تحقق:
- approval steps skipped
- pending restored
- withdrawn event emitted
- no further approval possible

D) Auto Approval Flow
policy.requiresApproval = false

تحقق:
- status مباشرة approved
- no approval steps created
- leave.requested + leave.approved emitted
- used increment مباشرة

E) Invalid Transitions

اختبر:
- approve rejected request
- reject approved request
- withdraw approved request
- double approval
- double rejection

المطلوب:
- proper 4xx responses
- no state corruption
