عنوان المهمة:
WORKFLOW PHASE 14-C — Platform Access Boundary & Route Guards

المهمة الحالية:
تثبيت حدود الوصول داخل واجهة Platform Administration بحيث تكون الصفحات، التبويبات، الأزرار، والإجراءات متوافقة مع صلاحيات P14-B.

الهدف:
منع تجربة "زر ظاهر ثم backend يرفض" قدر الإمكان، مع بقاء backend هو المصدر النهائي للحماية.

بمعنى:
- Backend authorization remains source of truth
- Frontend hides/disables unauthorized UI
- Routes redirect أو تعرض Access Denied حسب الصلاحية
- Navigation sidebar يصير permission-aware
- Tenant console tabs/actions تصير permission-aware
- Platform users page actions تصير permission-aware

أضف components:
- PlatformPermissionRoute
- PlatformAccessDenied

PlatformPermissionRoute props:
- permission
- anyOf
- allOf
- children

Behavior:
- loading state while current platform user loads
- if not platform user → redirect /sign-in أو access denied حسب pattern الحالي
- if missing permission → PlatformAccessDenied
- if allowed → render children

PlatformAccessDenied:
- English + Arabic short message
- shows required permission code
- no sensitive data
- no stack trace
