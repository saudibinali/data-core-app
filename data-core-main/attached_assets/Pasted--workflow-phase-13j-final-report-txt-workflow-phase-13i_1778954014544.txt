WORKFLOW PHASE 14-A — Platform User Account Creation Foundation + Root Protection Policy

المهمة الحالية:
تأسيس إدارة مستخدمي المنصة الداخليين داخل Platform Administration، مع حماية صارمة لحساب مالك المنصة الجذري.

الهدف:
تمكين Platform Owner / authorized platform admin لاحقًا من:
- إنشاء Platform User داخلي
- عرض قائمة Platform Users
- عرض تفاصيل Platform User
- تغيير حالة Platform User بشكل مضبوط
- ربط المستخدم بدور مبدئي
- منع إنشاء أو تعيين Root من الواجهة
- حماية Root Platform Owner من التعديل أو التعطيل أو تغيير كلمة المرور من لوحة الإدارة

بطريقة:
controlled platform user foundation
بدون permission matrix كاملة وبدون tenant users وبدون reset password وبدون email invite sending

3 — ROOT PLATFORM OWNER PROTECTION POLICY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

أضف policy/lib:

root-platform-owner-policy.ts

ROOT_PLATFORM_OWNER_PROTECTION_POLICY:
- root_platform_owner — مالك المنصة الجذري
- protected_account — حساب محمي
- immutable_role — الدور غير قابل للتغيير
- non_deletable — غير قابل للحذف
- non_disableable — غير قابل للتعطيل
- non_lockable — غير قابل للقفل
- password_reset_blocked_from_admin_ui — منع إعادة تعيين كلمة المرور من لوحة الإدارة
- email_change_blocked — منع تغيير البريد من لوحة الإدارة
- self_promotion_blocked — منع ترقية النفس
- root_role_assignment_blocked — منع تعيين دور Root من الواجهة
- cannot_manage_equal_or_higher_privilege — منع إدارة مستخدم بنفس أو أعلى من صلاحيتك
- cannot_disable_last_root_owner — منع تعطيل آخر مالك جذري
- requires_break_glass_recovery — يتطلب إجراء طوارئ للاسترداد لاحقًا
- audit_required — كل محاولة ممنوعة تسجل

كلها true.

Exports:
- isRootPlatformOwner(user)
- isProtectedPlatformAccount(user)
- canAssignPlatformRole(actor, targetRole)
- canManagePlatformUser(actor, targetUser)
- canChangePlatformUserStatus(actor, targetUser, nextStatus)
- canResetPlatformUserPasswordFromAdmin(actor, targetUser)
- canChangePlatformUserEmail(actor, targetUser)
- validatePlatformUserCreate(actor, payload)
- validatePlatformUserStatusChange(actor, targetUser, nextStatus)
- buildBlockedPlatformUserActionAuditEvent(actor, targetUser, action, reason)

Rules:
- لا يمكن إنشاء root_platform_owner من API العادي
- لا يمكن تعيين root_platform_owner من role update العادي
- لا يمكن actor يرقّي نفسه
- لا يمكن actor يدير مستخدم بنفس أو أعلى من صلاحياته
- لا يمكن تعطيل/قفل/حذف root protected account
- لا يمكن تغيير بريد root من admin UI
- لا يمكن reset password للـ root من admin UI
- كل blocked attempt يسجل audit

مهم:
لا تنفذ break-glass الآن.
فقط policy + guard + audit.
