WORKFLOW PHASE 13-C — Subscription Metadata, Trial Windows & Renewal Lifecycle Foundations

- ضبط planCode

بدون أي payment provider أو billing automation

3 — SUBSCRIPTION STATUS MODEL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ثبّت SubscriptionStatus:

- trialing
- active
- renewal_due
- grace_period
- expired
- suspended
- cancelled
- unknown

مع deterministic semantics:

trialing:
trialEndsAt في المستقبل

active:
billingPeriodEnd في المستقبل وليس قريب جدًا من renewalDueAt

renewal_due:
billingPeriodEnd قريب أو renewalDueAt وصل

grace_period:
billingPeriodEnd انتهى لكن gracePeriodEndsAt في المستقبل

expired:
billingPeriodEnd انتهى و gracePeriodEndsAt انتهى أو غير موجود

suspended:
subscription suspended إداريًا

cancelled:
cancelledAt موجود أو status cancelled

unknown:
لا توجد subscription metadata

مهم:
لا تعمل automatic workspace suspension في P13-C.

Validation:
- planCode valid
- dates ISO valid
- billingPeriodStart < billingPeriodEnd
- trialStartedAt <= trialEndsAt
- gracePeriodStartedAt <= gracePeriodEndsAt
- cancelledAt cannot be before createdAt if available
- status must be known
- no impossible combinations مثل active مع cancelledAt
