3 — SYNCHRONOUS DOMAIN VALIDATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

داخل POST /hr/leave-requests:

نفّذ validations حقيقية قبل INSERT:

A) Balance validation
- اقرأ hr_leave_balances
- تأكد:
entitled - used - pending >= daysRequested
- استخدم locking أو transaction strategy تمنع race conditions

B) Conflict validation
- منع overlapping leave requests لنفس الموظف
- يشمل pending + approved

C) Date validation
- startDate <= endDate
- businessDaysCount > 0
- policy eligibility الأساسية

D) Business-day calculation
- استخدم:
hr_work_calendars
hr_calendar_holidays
- احسب businessDaysCount فعليًا
- لا تعتمد على client input
