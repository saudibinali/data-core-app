2. HIERARCHY & APPROVAL TRUTH
================================================================================

employee hierarchy real?
  PARTIAL. directManagerId on employees supports tree IF populated.
  No DB FK, no UI maintenance on detail, no sync with users.lineManagerId.

org structure real?
  PARTIAL. hr_org_units tree is real in schema/API.
  Legacy departments flat model still drives users/workflows.
  No org chart; dual directories.

approvals operable?
  YES WITH CAVEATS. Leave uses employee manager + admin fallback.
  Workflows use user lineManagerId — different chain.
  Incomplete structure → wrong approver or skipped steps.

workflow hierarchy scalable?
  LOW-MEDIUM. Workflow engine can target "manager" but reads wrong table
  for HCM tenants. department_head not implemented.
  Cannot scale org-based routing without consolidation.

3. MANDATORY vs OPTIONAL vs RISKS
================================================================================

SHOULD BECOME MANDATORY (enterprise target — not current state):
  • employees.orgUnitId (or equivalent) at hire completion
  • employees.directManagerId for non-top-level roles
  • jobTitleId OR positionId (once wired)
  • employees.userId for self-service-enabled workspaces
  • Single canonical org tree (deprecate or sync legacy departments)

SHOULD REMAIN OPTIONAL (valid cases):
  • userId for non-login workforce (contractors, pre-boarding)
  • endDate / probation for permanent staff
  • custom fields per workspace policy
  • Multiple user departments IF matrix org formally supported (not today)

ARCHITECTURAL RISKS:
  • Dual org + dual manager — reporting and approvals disagree
  • positionId schema without application wiring
  • Position history without employee row update
  • Admin fallback approver masks data quality issues
  • No tree validation on org unit parentId

FAKE FLEXIBILITY:
  • "Organization structure" entitlement vs optional orgUnitId
  • hr_positions headcount/occupancy without assignment
  • Job movements tab implying transfer workflow without state update
  • Workflow department_head approver type
  • Text company/branch fields parallel to structured org units

4. USER MANAGEMENT vs ENTERPRISE WORKFORCE PLATFORM
================================================================================

Answer: CLOSER TO **USER MANAGEMENT + HR MODULE ADD-ON** than unified
enterprise workforce platform.

Evidence:
  • users table predates / parallels employees
  • Workflows, tickets, legacy departments anchor on users
  • HCM org tree serves HR routes but not platform-wide
  • No enforced workforce invariants
  • Strong employee schema + weak governance = "metadata-rich, runtime-soft"

Maturity estimate:
  User / access directory .............. 70%
  HR employee records .................... 65%
  Organizational structure (unified) ..... 35%
  Reporting / manager hierarchy .......... 40%
  Enterprise HCM governance .............. 25%
