ITEM 1 — Global Synchronous Error Boundary in EventBus.fanOut()
  File: lib/core-events/src/bus.ts
  Problem:
    Promise.allSettled was called with the result of listeners.map(), but if a
    listener's handler() threw *synchronously* (before returning a Promise), the
    throw escaped the .map() callback and propagated up to the await site.
    Promise.allSettled never received that element — the settled array would be
    shorter than the listeners array, silently corrupting the error-accounting
    loop.  Other listeners were still at risk of being counted incorrectly.
    This was a real risk: any listener that forgot to `async` its function, or
    threw before a first `await`, would trigger the bug.
  Fix:
    Each entry.handler(event) call is now wrapped in a try/catch before being
    passed to Promise.allSettled.  Sync throws are converted to Promise.reject(),
    which Promise.allSettled handles at the correct index.  The existing error
    logging and EmitResult.errors collection handles them identically to async
    rejections.  The emit() API and EmitResult shape are completely unchanged.
  Code change:
    BEFORE:
      const settled = await Promise.allSettled(
        listeners.map((entry) => entry.handler(event)),
      );
    AFTER:
      const settled = await Promise.allSettled(
        listeners.map((entry) => {
          try {
            return entry.handler(event);
          } catch (syncErr) {
            return Promise.reject(syncErr instanceof Error ? syncErr : new Error(String(syncErr)));
          }
        }),
      );


ITEM 2 — BUG-001 Fixed: leave.requested duplicate in registry.ts
  File: artifacts/api-server/src/lib/events/registry.ts
  Problem:
    "leave.requested" was registered TWICE in the event catalog:
      Entry 1 (line ~472): module = "hr", correct schema
        fields: employeeId, leaveType (enum), startDate (date), endDate (date)
      Entry 2 (line ~630): module = "forms", diverged schema
        fields: leave_type (snake_case vs camelCase), no dates, extra "emergency"
        enum value, DEPARTMENT_FIELD.  Missing employeeId.
    The duplicate caused:
      • Potential duplicate rows in platformEventRegistryTable on seed/upsert.
      • Schema drift — field names inconsistent between both entries.
      • Workflow builder showing "leave.requested" twice in the trigger picker,
        confusing workspace admins selecting the event.
    Root cause: copy-paste error when a forms-based leave workflow was added.
  Fix:
    The forms duplicate (Entry 2) has been REMOVED.
    Canonical owner: module = "hr" (Entry 1, unchanged).
    A comment block has been added at the removal site explaining the root cause,
    the canonical owner, and the correct pattern for future dual-event scenarios.
    Note: "forms.form.submitted" already covers generic form submissions from
    the forms module.  A leave form submitted via the form builder correctly
    fires form.submitted.  leave.requested is a distinct HR domain event fired
    by the HR route when an official leave request is recorded.

REMAINING RISKS (inherited, not introduced in this sprint)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  RISK-01  In-memory IdempotencyGuard — cleared on restart.
           Impact: if the server restarts mid-emit (e.g. deploy), a partially
           processed event may be reprocessed.
           Mitigation: acceptable at current scale.
           Fix path: persist event IDs to DB table with TTL.

  RISK-02  No dead-letter queue — failed listeners are logged and dropped.
           A permanent DB failure in a listener drops the side-effect silently.
           Fix path: add a failed_events table + background retry worker.

  RISK-03  Synchronous bridge (await dispatch) adds latency to bus fanout.
           All bus listeners wait for bridge → eventDispatcher → DB write.
           Impact: bus emit() resolves ~10-30ms slower than a pure in-memory bus.
           Mitigation: routes use `void appEventBus.emit()` — never blocks HTTP.
           Fix path: make bridge fire-and-forget when scale requires it.

  RISK-04  notifications.ts import in index.ts (no-op scaffold).
           Cosmetic only — file is intentional and documented.
           Fix path: one-line removal when convenient.

  RISK-05  Invitation-flow users (POST /users sync) do NOT emit EMPLOYEE_CREATED.
           Activity logs and admin notifications not triggered for invited users.
           Fix path: add appEventBus.emit(EMPLOYEE_CREATED) to the user sync route.
