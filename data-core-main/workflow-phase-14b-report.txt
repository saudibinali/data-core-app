artifacts/api-server/src/middlewares/requireAuth.ts
  - Added platformRoleCode?: string | null to AuthRequest interface
  - Added isRootOwner?: boolean to AuthRequest interface
  - requireAuth now selects platformRoleCode + isRootOwner from DB, attaches to req
  - Added requirePlatformPermission(permissionCode) factory function:
      - Requires role === "super_admin" (platform user only)
      - Calls hasPlatformPermission from platform-permissions.ts
      - On denial: req.log.warn + fire-and-forget audit write to activityLogsTable
      - Returns 403 with { code: "PERMISSION_DENIED", required: permissionCode }
      - Legacy root (platformRoleCode IS NULL) gets all permissions automatically
  - Added requireAnyPlatformPermission(permissionCodes[]) factory function
  - requireSuperAdmin kept intact for backward compat

artifacts/api-server/src/routes/platform-users.ts
  - Import changed: requireSuperAdmin → requirePlatformPermission
  - GET /platform/users → requirePlatformPermission("platform.users.read")
  - GET /platform/users/:userId → requirePlatformPermission("platform.users.read")
  - POST /platform/users → requirePlatformPermission("platform.users.create")
  - PATCH /platform/users/:userId/status → requirePlatformPermission("platform.users.status.update")
  - NEW: PATCH /platform/users/:userId/role → requirePlatformPermission("platform.users.role.update")
    - Validates: roleCode is assignable (not root), reason ≥ 10 chars, confirmation = true
    - Checks: target is not root/protected (isProtectedPlatformAccount)
    - Checks: canManagePlatformUser (actor can manage target)
    - Checks: canAssignPlatformRole (actor's privilege level allows assigning roleCode)
    - Writes audit log on every blocked/success outcome
    - Returns { user, previousRoleCode, nextRoleCode }
