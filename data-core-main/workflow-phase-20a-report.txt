CURRENT AUDIT SUMMARY
--------------------------------------------------------------------------------
- Core: hr_attendance (1 row/employee/day), hr_shifts, hr_work_calendars,
  hr_calendar_holidays; times as HH:MM text; source_type enum without ingest.
- APIs: ~40 endpoints in hr.ts (CRUD, import preview/confirm, export, OT).
- UI: hr-attendance.tsx (/admin/hr/attendance); no self-service clock UI.
- Leave: canonical leave_requests does NOT update attendance; dual leave UI.
- Payroll: does NOT read attendance; payslip day fields unused.
- Reports: hr.attendance.period via P19 reporting + Report Center (GO).
- Import: inline XLSX flow; NOT on import_jobs; confirm drops preview fields.
- Events: no attendance types on appEventBus (leave only).

--------------------------------------------------------------------------------
PROPOSED CANONICAL ARCHITECTURE
--------------------------------------------------------------------------------
Workforce Event Platform layers:
  Sources → Connectors → raw_events → attendance_events → daily_summaries
  → dual-write hr_attendance (transition) → payroll/reports/leave consumers

11 new logical models specified (sources, devices, integrations, raw_events,
events, daily_summaries, adjustments, geofences, policies, sync_jobs,
import_batches) — design only, no migrations.

--------------------------------------------------------------------------------
INTEGRATION STRATEGY
--------------------------------------------------------------------------------
Vendor-agnostic AttendanceConnector interface; webhook + poll modes;
encrypted credentials; idempotency; mapping profiles; DB sync workers
(no Redis); generic_webhook + vendor adapters as plugins.

--------------------------------------------------------------------------------
GPS STRATEGY
--------------------------------------------------------------------------------
Geofence circle per work location; punch-time-only location capture;
privacy/retention rules; spoofing flags; offline queue documented.
Implementation BLOCKED until P20-D+.

--------------------------------------------------------------------------------
IMPORT STRATEGY
--------------------------------------------------------------------------------
Migrate to import_jobs + attendance_import_batches + Document Registry;
server-side parse; dry-run; mapping profiles; duplicate detection;
soft revert via adjustment tagging. Legacy importer remains until flag cutover.

--------------------------------------------------------------------------------
NORMALIZATION
--------------------------------------------------------------------------------
Event types (clock_in/out, overlays); source priority; late/early/OT
calculation from shifts; missing punch rules; timezone/local_date;
conflict resolution; rebuild-day capability.

--------------------------------------------------------------------------------
PAYROLL / LEAVE INTERACTION
--------------------------------------------------------------------------------
Target: payroll READS summaries; leave generates leave_overlay events;
holiday overlays; period locks before payroll close. No payroll redesign
in P20-A. Current coupling: BLOCKED.

--------------------------------------------------------------------------------
SECURITY / COMPLIANCE
--------------------------------------------------------------------------------
Location minimization; no biometric storage in ERP; append-only raw events;
workspace isolation; audit via adjustments + import_jobs; retention policy;
anti-tamper via locks and idempotency.
