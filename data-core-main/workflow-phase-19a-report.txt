EXECUTIVE SUMMARY
================================================================================
P19-A established enterprise-grade architecture for workspace-scoped document
storage, communication (email + notifications), reporting/export, and import
processing. The foundation extends P18 workspace isolation (workspace_id) and
avoids duplicate portals per p18-a-do-not-duplicate-list.md.

Current platform has strong in-app notifications (event bus + SSE) and ad-hoc
HR Excel import/export, but lacks unified document registry, per-workspace SMTP,
async job queues, PDF report generation, and secured upload ACLs.

No migrations, SMTP rollout, storage migration, or file engine code was added.

CANONICAL INFRASTRUCTURE DIRECTION
================================================================================
Single workspace document registry (documents + document_versions) replacing
fragmented object_path / storage_path / JSONB attachment patterns.

Communication pipeline: Event → Notification Engine → Queue → Template Engine →
Workspace SMTP / In-app → delivery tracking → audit.

Job tables: notification_jobs, import_jobs, export_jobs, generated_reports.

Extend notifications table with workspace_id (do not create parallel inbox).

================================================================================
WORKSPACE SMTP STRATEGY
================================================================================
Per-workspace workspace_smtp_configs (encrypted credentials, verified senders).
Platform env SMTP remains for system mail only.
Migrate from lib/email.ts env-only pattern to WorkspaceMailer + PlatformMailer.
Wire or deprecate platform_settings smtp category to avoid split-brain.

================================================================================
DOCUMENT ARCHITECTURE SUMMARY
================================================================================
Lifecycle: Upload → Validate/Scan → Store → Version → Access → Export → Archive
Storage key: {workspace_id}/{domain}/{entity}/{document_id}/{version}
Unify: hr_employee_documents, form_submission_files, commercial PDFs, leave
attachments (future migration).
AI hooks documented only (OCR, parsing) — not implemented.

================================================================================
REPORTING / EXPORT STRATEGY
================================================================================
Report definitions registry + async export_jobs for large datasets.
Excel: extend existing xlsx HR patterns.
PDF: new engine (puppeteer or service) for payroll/compliance — phase P19-B+.
Scheduled reports via communication layer.

Activate reports.view permission with real /reports product in later phase.
