ARCHITECTURE NOTES
────────────────────────────────────────────────────────────────────────────────

• Pure derivation: tenant-health-intelligence.ts is side-effect-free and
  deterministic. All state comes from the TenantHealthInput parameter.

• Layered composition: health derives from signals produced by prior phases
  (P13-C subscription, P13-E usage, P13-F renewal, P13-B lifecycle).

• Fails closed: missing subscription data yields subscriptionStatus = "unknown",
  which surfaces "subscription_unknown" signal and prevents silent green health.

• No enforcement path: the health system has no write surface. The UI renders
  informational badges and advisory actions only. The safety contract is verified
  at module load time via import-time throw guard.

• Component breakdown: 6 component health cards give admins actionable clarity
  on which dimension (lifecycle / subscription / renewal / usage /
  entitlements / governance) is degraded.
