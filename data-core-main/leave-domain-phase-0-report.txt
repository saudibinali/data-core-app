  leave.requested — LeaveRequestedPayload
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ leaveRequestId:   number          FK to leave_requests.id              │
  │ employeeUserId:   number          the requesting employee's user ID     │
  │ leaveType:        string          annual|sick|unpaid|maternity|         │
  │                                   emergency|other                       │
  │ startDate:        string          ISO date "YYYY-MM-DD"                 │
  │ endDate:          string          ISO date "YYYY-MM-DD"                 │
  │ daysRequested:    number          business days (excl. weekends+holidays)│
  │ departmentId?:    number|null                                           │
  │ employeeNote?:    string          reason from employee                  │
  │ leavePolicyId?:   number|null     FK to hr_leave_policies               │
  │ requiresApproval: boolean         from policy (drives approval chain)   │
  └─────────────────────────────────────────────────────────────────────────┘

  leave.approved — LeaveApprovedPayload
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ leaveRequestId:   number                                                │
  │ employeeUserId:   number                                                │
  │ leaveType:        string                                                │
  │ startDate:        string                                                │
  │ endDate:          string                                                │
  │ daysApproved:     number          stored at creation, not recomputed    │
  │ approvedByUserId: number          final approver in the chain           │
  │ leavePolicyId?:   number|null                                           │
  │ departmentId?:    number|null                                           │
  └─────────────────────────────────────────────────────────────────────────┘

  leave.rejected — LeaveRejectedPayload
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ leaveRequestId:   number                                                │
  │ employeeUserId:   number                                                │
  │ leaveType:        string                                                │
  │ startDate:        string                                                │
  │ endDate:          string                                                │
  │ daysRequested:    number                                                │
  │ rejectedByUserId: number          the rejecting approver                │
  │ rejectionReason?: string          optional note                         │
  │ departmentId?:    number|null                                           │
  └─────────────────────────────────────────────────────────────────────────┘

  leave.cancelled — LeaveCancelledPayload
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ leaveRequestId:    number                                               │
  │ employeeUserId:    number                                               │
  │ leaveType:         string                                               │
  │ startDate:         string                                               │
  │ endDate:           string                                               │
  │ cancelledByUserId: number         the HR/admin who cancelled            │
  │ wasApproved:       boolean        true → balance + attendance restore   │
  │ departmentId?:     number|null                                          │
  └─────────────────────────────────────────────────────────────────────────┘

  leave.withdrawn — LeaveWithdrawnPayload
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ leaveRequestId:   number                                                │
  │ employeeUserId:   number          (IS the withdrawer — employee only)   │
  │ leaveType:        string                                                │
  │ startDate:        string                                                │
  │ endDate:          string                                                │
  │ daysRequested:    number                                                │
  │ departmentId?:    number|null                                           │
  └─────────────────────────────────────────────────────────────────────────┘

  leave.balance_adjusted — LeaveBalanceAdjustedPayload
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ leaveBalanceId:    number         FK to hr_leave_balances.id            │
  │ employeeId:        number         FK to employees.id                    │
  │ employeeUserId:    number         FK to users.id                        │
  │ leaveType:         string                                               │
  │ year:              number         e.g. 2026                             │
  │ adjustmentDays:    number         signed: + grant, - deduction          │
  │ reason:            string         required for audit trail              │
  │ adjustedByUserId:  number         the HR user who made the adjustment   │
  │ leavePolicyId?:    number|null                                          │
  └─────────────────────────────────────────────────────────────────────────┘

  Payload design principles applied:
    ✓ Only domain facts — no workflow state, no UI fields
    ✓ Consistent naming: camelCase, entity.action pattern
    ✓ All dates as ISO date strings (not timestamps)
    ✓ Boolean fields for binary state (wasApproved, requiresApproval)
    ✓ Signed numbers for adjustments (adjustmentDays)
    ✓ Optional fields use ? (nullable FK fields use number|null)
    ✓ Reason/note fields: required where audit demands, optional where advisory
