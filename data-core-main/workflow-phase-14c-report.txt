SUMMARY
────────────────────────────────────────────────────────────────────────────────

P14-C implements client-side permission gating so that every page, navigation
item, tab, button, and write action in the Super Admin area is visible and
accessible only to platform users whose role grants the required permission.
The backend remains the authoritative enforcement layer (requirePlatformPermission
middleware from P14-B). P14-C adds a synchronous, zero-network-request client
layer on top using the same permission matrix.

All P14-C deliverables are complete and tested.

artifacts/ops-platform/src/lib/platform-access.ts
  - Pure client-side permission library — zero network, zero React, fully synchronous
  - MinimalPlatformUser interface: { role?, platformRoleCode?, isRootOwner? }
  - getEffectivePlatformRoleCode(user): resolves legacy root + isRootOwner flag
  - getCurrentPlatformPermissions(user): returns Set<PlatformPermissionCode>
  - hasPlatformPermissionClient(user, code): boolean
  - hasAnyPlatformPermissionClient(user, codes[]): boolean
  - hasAllPlatformPermissionsClient(user, codes[]): boolean
  - canViewPlatformNavItem(user, navKey): boolean — controls sidebar visibility
      NavItemKey values: "overview" | "workspaces" | "tenant-registry" |
        "platform-users" | "platform-activity" | "event-log" | "platform-settings"
      Permission requirements per key:
        overview          — always visible (unrestricted)
        workspaces        — always visible (unrestricted)
        tenant-registry   — tenants.read
        platform-users    — platform.users.read
        platform-activity — platform.activity.read
        event-log         — audit.read
        platform-settings — platform.settings.read
  - canViewTenantConsoleTab(user, tab): boolean — controls TenantAdminConsole tabs
      Tab values: overview | lifecycle | subscription | entitlements |
                  usage | renewal | health | evaluation
      Permission requirements per tab:
        overview      — tenants.read OR health.read OR usage.read OR renewal.read
        lifecycle     — tenants.lifecycle.update OR tenants.read
        subscription  — subscriptions.read
        entitlements  — entitlements.read
        usage         — usage.read
        renewal       — renewal.read
        health        — health.read
        evaluation    — evaluation.read
  - canPerformPlatformAction(user, actionKey): boolean — controls write buttons
      PlatformActionKey values (6 total — no forbidden scope):
        platform.user.create                → platform.users.create
        platform.user.status.update         → platform.users.status.update
        platform.user.role.update           → platform.users.role.update
        tenant.lifecycle.update             → tenants.lifecycle.update
        tenant.subscription.update          → subscriptions.update
        tenant.entitlement.override.update  → entitlements.override.update
  - canAccessPlatformRoute(user, routeKey): boolean — for route-level guards
      RouteKey values: "platform.users" | "tenant.registry" |
                       "platform.activity" | "audit"
