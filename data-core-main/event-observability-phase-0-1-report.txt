PHASE 0-A: BS-01 — triggerEventLogId broken trace link (engine.ts)
  File:  artifacts/api-server/src/lib/workflows/engine.ts
  Fix:   WorkflowEngine wildcard listener now uses the full EventListener
         signature (payload, logId). The logId (workspace_event_logs.id) is
         passed to handleEvent() and written to triggerEventLogId on every
         workflow_executions INSERT.
  Before:
         trigger_event_log_id = NULL on every row
         SELECT * FROM workflow_executions WHERE trigger_event_log_id = X → 0 rows
  After:
         trigger_event_log_id = <logId> on every new row
         SELECT * FROM workflow_executions WHERE trigger_event_log_id = X → ✓ results
  Listener name "WorkflowEngine" now passed to eventDispatcher.on() as 3rd arg.

         Added idx_notifications_bus_event_id index.

         helpers now accept busEventId parameter and pass it to the DB.
         All 5 direct inserts in leave listeners (requested, approved, rejected,

         SELECT * FROM notifications WHERE bus_event_id = '<uuid>'

PHASE 1-D: Bridge + listener timing logs
  File:  artifacts/api-server/src/lib/events/bridge.ts
         artifacts/api-server/src/lib/events/listeners/activity.ts
         artifacts/api-server/src/lib/events/listeners/notifications-bus.ts
  Fix:   Every listener now records timing at DEBUG level:
           { listener: "<name>", eventId: "<uuid>", duration_ms: N, success: bool }
         Bridge records dispatch timing after every eventDispatcher.dispatch() call:
           { bridge: "bus→dispatcher", eventType: "...", eventId: "...", duration_ms: N }
  Log level: debug (not info) — high-volume, intended for log aggregators, not production streams.

PHASE 1-E: Named listener slots in dispatcher (dispatcher.ts)
  File:  artifacts/api-server/src/lib/events/dispatcher.ts
  Fix:   Changed internal storage from Map<string, EventListener[]> to
         Map<string, Array<{ name: string; fn: EventListener }>>.
         eventDispatcher.on() now accepts an optional 3rd arg listenerName:
           on(eventName: string, listener: EventListener, listenerName?: string): void
         Falls back to "<eventName>#<index>" if name omitted (backward compatible).
         Result stored in workspace_event_logs.result JSONB:
           { listeners: [{ name: "WorkflowEngine", success: true, durationMs: 12 }] }
         Partial failure semantics:
           All failed → status = "failed"
           Some failed → status = "completed" with errors in error column
  Backward compat: all existing callers that don't pass a name are unaffected.
