================================================================================
2) TENANT BILLING APIs
================================================================================

Router: artifacts/api-server/src/routes/tenant-billing.ts
Registered in artifacts/api-server/src/routes/index.ts

Endpoints (GET only):
  GET /api/tenant/billing/invoices
    - requireAuth + tenant.billing.invoices.read
    - Filters: status, from, to (invoiceDate)
    - Returns { invoices: TenantInvoiceSummary[] }
    - Excludes draft invoices from tenant view

  GET /api/tenant/billing/invoices/:invoiceId
    - requireAuth + tenant.billing.invoices.read
    - Returns { invoice: TenantInvoiceSummary }
    - 404 if not in current workspace or draft

  GET /api/tenant/billing/invoices/:invoiceId/document
    - requireAuth + tenant.billing.invoiceDocuments.download
    - Streams PDF via invoice-document-storage (same private store as P15-C)
    - Content-Disposition attachment; no public URL

Forbidden (no routes): POST, PATCH, DELETE, upload, status update, generate, payment, email.

Workspace isolation:
  - workspaceId from req.workspaceId (JWT/session DB lookup via requireAuth).
  - All queries filter eq(commercialInvoicesTable.workspaceId, workspaceId).
  - No tenantId/workspaceId from request body or query used for authorization.

3) DATA EXPOSURE RULES
================================================================================

Tenant API response fields (toTenantInvoiceSummary):
  - invoiceNumber, invoiceTitle, invoiceDate, dueDate
  - invoiceAmount, currency
  - billingPeriodStart, billingPeriodEnd
  - status
  - documentAvailable, documentFileName, uploadedAt

Hidden from tenant responses:
  - notes (internal platform)
  - createdBy, updatedBy
  - storageKey, checksum
  - externalAccountingReference, externalAccountingSystemName
  - commercialAccountId, contractTermId, workspaceId
  - audit metadata in response body (audit written to activity_logs only)

Draft invoices: excluded (platform-internal until issued/shared).

================================================================================
7) AUDIT EVENTS
================================================================================

Written to activity_logs (metadata JSON, no secrets):

  - tenant_invoice_viewed (list + detail success)
  - tenant_invoice_document_downloaded (PDF stream authorized)
  - tenant_invoice_access_denied (invoice not in workspace / not found)
  - tenant_invoice_document_download_blocked (no document / storage error)

Metadata includes: actorId, workspaceId, invoiceId, invoiceNumber (when known),
action, result, timestamp.
