OBJECTIVE
────────────────────────────────────────────────────────────────────────────────

P9-C delivers the governance API layer that surfaces the P9-A isolation engine
and P9-B workload partition engine as structured, workspace-scoped REST endpoints.
The phase introduces a new pure engine (tenant-governance.ts) that composes all
P8/P9 sub-results into a single TenantGovernanceView model, and wires it into
four new Express routes.

New file (~430 lines). Pure, deterministic, advisory-only.
No DB, no async, no cross-tenant leakage.

EXPORTS
  Types
    IsolationHealthStatus       "healthy" | "warning" | "elevated" | "critical"
    FairnessLevel               "fair" | "reduced" | "constrained" | "at_minimum"
    HotspotLevel                "none" | "low" | "moderate" | "high" | "critical"
    StarvationRisk              "none" | "low" | "moderate" | "high"
    SchedulerFairnessStatus     advisory scheduler weight + noisy-tenant summary
    PartitionPressureSummary    flattened P9-B partition for embedding
    HotspotSummary              P8-E concentration → classified hotspot level
    AdvisorySummary             P8-F signals → severity distribution
    TenantGovernanceView        full composed view (workspaceId-scoped)
    TenantGovernanceViewInput   input bundle for buildTenantGovernanceView()

  Functions
    buildTenantGovernanceView(input, context?)
      Scope guard: partition.workspaceId ≠ isoContext.workspaceId → throws
      TenantIsolationViolation(CROSS_WORKSPACE_ACCESS).
      Composes 5 sub-models → emits tenant_governance_view_generated event.

    computePartitionPressureSummary(partition)
      Pure projection from P9-B TenantWorkloadPartition.
      Copies pressureComponents (no aliasing).

    computeSchedulerFairnessStatus(partition)
      schedulerWeight → fairnessLevel (1.00/0.75/0.50/0.25 → fair/reduced/constrained/at_minimum).
      containmentStatus → starvationRisk (contained→none / at_risk→low / pressured→moderate / saturated→high).
      noisyBehaviorCodes array is copied (not aliased).

    classifyIsolationHealth(riskLevel)
      Total mapping of P9-A TenantRiskLevel → IsolationHealthStatus:
      low→healthy, moderate→warning, high→elevated, critical→critical.

    computeHotspotSummary(hotspot?)
      Absent input → safe zero defaults (hotspotLevel="none").
      classifyHotspotLevel() takes max severity across two dimensions:
        concentration (ratio ≥ 0.70→critical / ≥0.50→high / ≥0.30→moderate / ≥0.10→low)
        urgency (count ≥ 5→critical / ≥3→high / ≥2→moderate / ≥1→low)

    computeAdvisorySummary(signals?)
      Absent input → informational defaults, zero counts.
      Maps GovernanceAdvisoryLevel → AdvisoryPressureLevel:
        critical→critical / urgent→high / elevated→medium / advisory→low / informational→none
      Counts critical and high severity signals from the signals array.

    deriveOperationalPriority(hotspot?, supplied?, signals?)
      Explicit supplied value always wins.
      Fallback: hotspot urgentOrCriticalCount ≥ 3 → critical, ≥ 1 → urgent, dominant → elevated.
      Fallback: governance advisory level → OperationalPriority mapping.
      Final fallback: "informational".

    emitPartitionOverviewEvent(...)    — for workload-partition route
    emitFairnessVisibilityEvent(...)   — for fairness-status route
    emitIsolationHealthEvent(...)      — for isolation-health route

OBSERVABILITY EVENTS (4 total)

  tenant_partition_overview_requested — emitted by GET /governance/workload-partition

SAFETY INVARIANTS
  READ-ONLY        never mutates input, DB, or scheduler state
  TENANT-SCOPED    scope guard fails closed on workspaceId mismatch
  DETERMINISTIC    same inputs → same output (timestamps via context injection)
  NO CROSS-TENANT  output only contains own workspace data
  JSON-SAFE        all fields are plain JSON-serializable (no class instances)
  FAIL-CLOSED      scope ambiguity throws TenantIsolationViolation

ROUTES: artifacts/api-server/src/routes/workflows.ts
────────────────────────────────────────────────────────────────────────────────

Four new routes registered AFTER GET /workflows/governance/signals (P8-F)
and BEFORE GET /workflows/executions/failed.

GET /workflows/governance/tenant-overview
  Auth:      requireAuth + requirePermission("workflow.view")
  DB:        COUNT running/waiting_delay executions; SAMPLE last 10 execution IDs
  Engines:   buildTenantIsolationContext() → evaluateWorkloadContainment()
             → assessTenantIsolationRisk() → buildTenantGovernanceView()
  Response:  full TenantGovernanceView (JSON)
  Event:     tenant_governance_view_generated (from engine)

GET /workflows/governance/workload-partition
  Auth:      requireAuth + requirePermission("workflow.view")
  DB:        COUNT running/waiting_delay executions
  Engines:   buildTenantIsolationContext() → evaluateWorkloadContainment()
             → computePartitionPressureSummary() + computeSchedulerFairnessStatus()
  Response:  { workspaceId, partitionId, containmentStatus, pressureSummary, fairnessStatus, evaluatedAt }
  Event:     tenant_partition_overview_requested

GET /workflows/governance/fairness-status
  Auth:      requireAuth + requirePermission("workflow.view")
  DB:        COUNT running/waiting_delay executions
  Engines:   buildTenantIsolationContext() → evaluateWorkloadContainment()
             → computeSchedulerFairnessStatus()
  Response:  { workspaceId, fairnessStatus, containmentStatus, pressureTotal, evaluatedAt }
  Event:     tenant_fairness_visibility_accessed

GET /workflows/governance/isolation-health
  Auth:      requireAuth + requirePermission("workflow.view")
  DB:        SAMPLE last 20 execution IDs for boundary validation
             COUNT running/waiting_delay executions (for partition/pressure event fields)
  Engines:   buildTenantIsolationContext() → assessTenantIsolationRisk()
             → classifyIsolationHealth()
  Response:  { workspaceId, isolationHealth, overallRisk, per-dimension risks,
               findings[], sampledExecutions, evaluatedAt }
  Event:     tenant_isolation_health_evaluated

All 4 routes:
  • Fail closed: req.workspaceId check + TenantIsolationViolation → 403
  • Advisory-only: no writes, no scheduler mutation
  • Workspace-scoped: all DB queries filter by req.workspaceId

DESIGN DECISIONS
────────────────────────────────────────────────────────────────────────────────

1. Composite view as pure function
   TenantGovernanceView is built entirely from pre-fetched engine outputs. The
   route handler owns the DB queries; the engine is pure and testable in isolation.

2. Optional P8-E/P8-F inputs
   The governance view gracefully handles absent hotspot/signals data with safe
   defaults. Lightweight routes (workload-partition, fairness-status) skip the
   heavier P8-E/P8-F computations for lower latency.

3. Route-level observability events (not engine-level) for endpoint-specific events
   tenant_governance_view_generated is engine-emitted (all callers benefit).
   The three endpoint-specific events (partition_overview, fairness_visibility,
   isolation_health) are emitted from route handlers so they carry HTTP context.

4. Scope guard location
   The scope guard (partition.workspaceId ≠ isoContext.workspaceId) lives at the
   entry of buildTenantGovernanceView, not at the route handler. This ensures any
   caller — including future non-HTTP callers — gets the same safety guarantee.

5. validateAnalyticsScope is throwing (not returning valid:false)
   Consistent with P9-A patterns: scope violations throw TenantIsolationViolation,
   not return a `valid: false` union. The P9-C tenant-overview route wraps the call
   in try/catch and logs the violation without aborting the request, since execution
   samples from the own workspace will never violate scope in practice.
