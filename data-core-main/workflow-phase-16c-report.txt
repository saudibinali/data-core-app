Rules:

  - limitValue >= 0 when set; null = unlimited

3. USAGE RESOLVER (read-only)
================================================================================

File: artifacts/api-server/src/lib/workspace-quota-resolver.ts

  resolveWorkspaceQuotaUsage(workspaceId)

Returns per quota:
  quotaKey, limitValue, currentUsage, usagePercent, status
  (ok|warning|exceeded|unlimited|unknown), warningThresholdPercent,
  isHardLimit, source, quotaLimitId, effectiveFrom, effectiveUntil

Usage sources (P16-C):
  - users.max → users count
  - employees.max → employees count
  - branches.max → hr_org_units type=branch
  - documents.max → hr_employee_documents count
  - storage.gb → sum(fileSize) / GB (approximate)
  - workflows.max → workflow_definitions (non-deleted)
  - custom.roles.max → workspace_custom_roles count
  - integrations.max, api.requests.monthly, ai.actions.monthly, reports.max → unknown

Behavior:
  - No mutations, blocks, or enforcement
  - Default catalog limit when no DB row; DB row with limitValue=null → unlimited

================================================================================
6. VALIDATION
================================================================================

  - Tenant must exist
  - subscriptionId must belong to same workspace
  - quotaKey must exist in catalog
  - limitValue >= 0 (null = unlimited)
  - warningThresholdPercent 1..100
  - effectiveFrom <= effectiveUntil
  - reason required when reducing limitValue or enabling isHardLimit (min 10 chars)
  - isHardLimit only when catalog.hardLimitSupported
  - Forbidden payload keys: payment, Stripe, checkout, tax, ledger, etc.
