2. DATABASE MODEL
================================================================================

Table: workspace_entitlements (lib/db/src/schema/workspace-entitlements.ts)

Fields:
  id, workspaceId, subscriptionId (optional), moduleKey, featureKey,
  isEnabled, source, effectiveFrom, effectiveUntil, reason, internalNotes,
  createdBy, updatedBy, createdAt, updatedAt

Rules:
  - featureKey empty string in DB = module-level entitlement (API exposes null)
  - unique (workspaceId, moduleKey, featureKey)
  - no hard delete — isEnabled=false / upsert only
  - optional link to workspace_subscriptions (P16-A)
  - coexists with legacy tenant_entitlement_overrides (P13-D)

5. HELPER BEHAVIOR (read-only)

================================================================================
7. VALIDATION
================================================================================

  - Tenant must exist
  - subscriptionId must belong to same workspace if provided
  - moduleKey must be in catalog
  - featureKey must belong to moduleKey when set
  - core cannot be set isEnabled=false (blocked + audit)
  - effectiveFrom <= effectiveUntil
  - reason required when disabling (min 10 chars)
  - forbidden payment/Stripe/card/tax payload keys rejected

================================================================================

Metadata: actorId, workspaceId/tenantId, entitlementId, moduleKey, featureKey,
previousEnabled, nextEnabled, source, reason, timestamp.
