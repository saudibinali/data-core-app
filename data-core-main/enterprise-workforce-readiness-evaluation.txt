3. FOUNDATION ARCHITECTURE TRUTH
================================================================================

REAL FOUNDATION (operational hooks):
  • hr_org_units + employees.orgUnitId
  • hr_job_titles / hr_job_grades + employee FKs
  • hr_leave_policies + leave requests + balances
  • hr_work_locations (underused)
  • POST /hr/foundation/seed

METADATA / LOOKUP ONLY:
  • hr_employee_statuses (employees.status text parallel)
  • hr_employment_types (employees.employmentType text parallel)
  • hr_contract_types (contracts.contractType text parallel)
  • hr_document_types (documents.documentType text parallel)
  • hr_probation_policies (no employee FK)
  • hr_positions seats (employees.positionId unwired)
  • Leave policy annualDays/accrualType (no accrual engine)

NOT ORGANIZATIONAL INTELLIGENCE:
  • No org graph traversal for approvals
  • No inheritance
  • No unit head resolver
  • No position incumbent lifecycle

================================================================================
4. WHAT SHOULD CHANGE LATER (PLANNING — NOT EXECUTED)
================================================================================

REDESIGN (targeted):
  • Unified org model (deprecate or sync legacy departments)
  • Position seat ↔ employee incumbent binding
  • Single ApproverResolutionService (employee graph + org heads)
  • FK foundation lookups on employee/contract/document rows

SIMPLIFY:
  • Remove or hide unwired foundation tabs until bound
  • Consolidate API under /hr/foundation/*
  • Align UI terminology (Position vs Title vs Role)
  • Single "Department" concept in admin UX

CONVERT TO RUNTIME ENTITIES:
  • hr_positions + occupancy from assignments
  • hr_leave_policies → accrual worker using annualDays/accrualType
  • hr_probation_policies → auto probationEndDate on hire
  • hr_employee_statuses → FK or enforced code sync

MAKE MANDATORY (enterprise tier):
  • orgUnitId on employee activation
  • directManagerId (except defined root roles)
  • jobTitleId or positionId
  • userId link when self-service modules enabled

KEEP CONFIGURABLE:
  • Leave policy catalog per workspace
  • Custom field definitions
  • Document type catalog (once enforced)
  • Work location catalog
  • Display order / bilingual labels
