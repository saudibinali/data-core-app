================================================================================
1) INVOICE RECORDS MODEL
================================================================================

Table: commercial_invoices
Schema: lib/db/src/schema/commercial-invoices.ts

Fields:
  - id, workspaceId, commercialAccountId, contractTermId (optional)
  - invoiceNumber (required, unique per workspace)
  - invoiceTitle, invoiceDate, dueDate
  - invoiceAmount, currency
  - billingPeriodStart, billingPeriodEnd
  - status: draft | issued | shared | paid | overdue | cancelled
  - externalAccountingSystemName, externalAccountingReference (optional)
  - notes
  - createdBy, updatedBy, createdAt, updatedAt

Rules:
  - Multiple invoices per tenant allowed.
  - invoiceNumber unique per workspace (DB unique index).
  - No tax fields, no payment fields, no PDF binary in this table.
  - Cancelled status used instead of hard delete.

================================================================================
2) INVOICE DOCUMENTS MODEL
================================================================================

Table: commercial_invoice_documents
Schema: lib/db/src/schema/commercial-invoice-documents.ts

Fields:
  - id, invoiceId (unique — one primary PDF per invoice in P15-C)
  - fileName, originalFileName, fileSize, mimeType
  - storageKey (opaque private path)
  - checksum (optional SHA-256)
  - uploadedBy, uploadedAt, createdAt

Rules:
  - Re-upload replaces prior PDF (file deleted from storage, row updated).
  - No DELETE document API (by design).
  - application/pdf only.

================================================================================
3) FILE STORAGE APPROACH
================================================================================

Module: artifacts/api-server/src/lib/invoice-document-storage.ts

Private filesystem storage (not public URLs):
  - Root: INVOICE_PDF_STORAGE_DIR env or data/invoice-documents/
  - buildStorageKey(workspaceId, invoiceId) → tenants/{ws}/invoices/{id}/{uuid}.pdf
  - saveInvoicePdf(storageKey, buffer)
  - getInvoicePdfStream(storageKey) / getInvoicePdfBuffer
  - deleteInvoicePdfIfExists on replace

Upload parsing: artifacts/api-server/src/lib/parse-invoice-pdf-upload.ts (busboy)
  - Field name: file
  - PDF only, max 10MB, .pdf extension required

Download: GET .../document streams via protected API with
commercial.invoiceDocuments.read — no direct public URL.

Note: General object-storage presigned URLs (GCS) exist for forms but were not
used for invoices to keep all PDF access behind platform auth + audit.

5) BACKEND APIs
================================================================================

Router: artifacts/api-server/src/routes/commercial-invoices.ts
Registered: artifacts/api-server/src/routes/index.ts

GET    /platform/tenants/:tenantId/commercial-invoices
       ?status=&contractTermId=  — commercial.invoices.read
GET    /platform/tenants/:tenantId/commercial-invoices/:invoiceId
       — commercial.invoices.read; document metadata only if
         commercial.invoiceDocuments.read
POST   /platform/tenants/:tenantId/commercial-invoices
       — commercial.invoices.update (metadata only, no PDF)
PATCH  /platform/tenants/:tenantId/commercial-invoices/:invoiceId
       — commercial.invoices.update
PATCH  /platform/tenants/:tenantId/commercial-invoices/:invoiceId/status
       — commercial.invoices.update + reason (min 10 chars)
POST   /platform/tenants/:tenantId/commercial-invoices/:invoiceId/document
       — commercial.invoiceDocuments.upload (multipart PDF)
GET    /platform/tenants/:tenantId/commercial-invoices/:invoiceId/document
       — commercial.invoiceDocuments.read (protected download)

No DELETE routes. Forbidden payload keys (tax, payment, stripe, etc.) rejected.

================================================================================
7) VALIDATION
================================================================================

  - Tenant + commercial account required
  - contractTermId must belong to same tenant/account
  - invoiceNumber required; unique per workspace
  - invoiceDate <= dueDate; billingPeriodStart <= billingPeriodEnd
  - invoiceAmount >= 0; currency whitelist
  - status enum; status change requires reason
  - PDF: application/pdf, .pdf, max 10MB
  - Rejects tax/payment/card/stripe fields in JSON body
