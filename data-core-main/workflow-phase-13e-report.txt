# Phase 13-E — Usage Limits, Quotas & Capacity Intelligence

**Date:** 2026-05-16
**Status:** ✅ Complete
**Branch:** main

## Objective

Deliver a read-only **capacity intelligence foundation** for super-admins: view seat/storage/API/workflow usage versus plan limits, detect approaching and exceeded signals, and display capacity risk summaries — with no billing, enforcement, or suspension logic of any kind.

Current source types per metric:

|---|---|---|

#### `artifacts/api-server/src/lib/tenant-usage-intelligence.ts`
Pure derivation library — no DB, no HTTP, no side effects.

Key exports:
- `USAGE_APPROACHING_THRESHOLD` = `0.8` (80%)
- `USAGE_EXCEEDED_THRESHOLD` = `1.0` (100%)
- `deriveUsageLimitStatus(usage, limit, supportsComparison, nullableMeansUnlimited)` → `UsageLimitStatus`
- `calculateUsagePercentage(usage, limit)` → `number | null`
- `isUsageApproachingLimit(row)` / `isUsageLimitExceeded(row)` — row-level helpers
- `buildUsageMetricRows(rawUsage, entitlementProfile, now)` → `UsageMetricRow[]`
- `deriveCapacityRiskLevel(rows)` → `RiskLevel`

  | Condition | Risk Level |
  |---|---|
  | All rows unknown/not_applicable/unlimited | `"unknown"` |
  | All comparable rows normal | `"none"` |
  | 1 approaching | `"low"` |
  | 2+ approaching | `"medium"` |
  | 1 exceeded | `"high"` |
  | 2+ exceeded | `"critical"` |

- `deriveTenantUsageProfile(tenantId, workspaceId, rawUsage, entitlementProfile, now)` → `TenantUsageProfile`
- `summarizeUsageWarnings(rows)` → `{ warnings: string[] }` — human-readable warning strings for approaching/exceeded metrics

## Modified Files

### `artifacts/api-server/src/lib/tenant-registry.ts`

**`TenantUsageSummary`** — expanded with 3 new fields:
```typescript
usageWarningCount:  number;   // count of metrics with status "approaching"
usageExceededCount: number;   // count of metrics with status "exceeded"
capacityRiskLevel:  string;   // "none" | "low" | "medium" | "high" | "critical" | "unknown"
```

**`buildUsageSummary(userCount, now, opts?)`** — optional `opts` accepts seatLimit, storageLimit, apiLimit, documentsLimit, usageWarningCount, usageExceededCount, capacityRiskLevel. All default to 0 / "unknown".

**`deriveRiskSignalSummary(..., usageSignals?)`** — 5th optional parameter:
```typescript
usageSignals?: { usageLimitApproaching: boolean; usageLimitExceeded: boolean }
```
Now factors into `riskLevel`: exceeded → medium; approaching → low (both below suspended/disabled).

### `artifacts/api-server/src/routes/tenants.ts`

**Added `workflowDefinitionsTable`** to the `@workspace/db` destructure.

**Added `collectTenantRawUsage(workspaceId, entitlementLimits)`** — local async helper:
- Queries `usersTable` for active seat count (live DB)
- Queries `workflowDefinitionsTable` for active workflow count (live DB)
- Reads `audit_retention_days` from entitlement limits (configured)
- Sets `workspaces = 1` (derived)
- Returns all 10 others as `unavailable`

**Added `GET /platform/tenants/:tenantId/usage`** (super-admin only, read-only):
1. Validates `tenantId` — 400 if invalid, 404 if workspace not found
2. Fetches subscription + entitlement overrides
3. Derives entitlement profile via `deriveTenantEntitlementProfile`
4. Collects raw usage via `collectTenantRawUsage`
5. Derives full usage profile via `deriveTenantUsageProfile`
6. Computes warning strings via `summarizeUsageWarnings`
7. Logs with `req.log.info` (warningCount, exceededCount, unknownCount, capacityRiskLevel)
8. Returns structured JSON:
```json
{
  "usageProfile":              { ... TenantUsageProfile },
  "entitlementProfileSummary": { planCode, planTier, enabledCount, customOverrides, limits },
  "rawUsageSources":           { metricCode → { sourceType, notes } },
  "warnings":                  ["Approaching limit...", ...]
}
```

## Architecture Notes

- **Contract-first**: metric codes are derived from `FeatureLimitCode` (single source of truth); adding a new feature limit automatically adds a new usage metric slot.
- **Fail-safe defaults**: missing/unavailable data returns `"unknown"` status, not `"normal"` — the system never under-reports problems.
- **No auto-actions**: the usage profile is a pure read-only view. All riskLevel and status values are informational only — no enforcement, suspension, or notification is triggered by this system.
- **Incrementally extensible**: to add live tracking for `storage_gb`, change its `defaultSourceType` from `"unavailable"` to `"live_db"` in the registry and add the DB query to `collectTenantRawUsage`. The UI, tests, and risk derivation all pick it up automatically.
