11. REMAINING TOPOLOGY / READINESS UI GAPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

These gaps are intentionally deferred — they require backend work or are
out of scope for P12-F frontend-only foundations:

a) Topology diff comparison
   Currently a UI placeholder. Will become functional when:
   - GET /api/platform/governance/topology/diff endpoint is implemented
   - useGovernanceTopologyDiff() hook is available
   - Two snapshot selection UI is added (requires snapshot list endpoint)

b) Live layer-to-dimension linking
   The topology page does not yet cross-reference readiness dimensions
   against live layer boundary data. A future pass could highlight which
   readiness dimension is affected by each boundary warning.

c) Snapshot list and history
   Currently only the latest snapshot is displayed. A snapshot history
   list (paginated GET) would enable diff selection and trend display.

d) Readiness score data
   The dimension grid renders score bars when score: number is present
   in the API response. Most dimensions will show "Unknown" until the
   backend computes per-dimension readiness scores (not yet implemented).

e) Real-time boundary push
   Topology and boundary data is fetched via polling (GOVERNANCE_QUERY_CONFIG
   staleTime). A WebSocket or SSE push for boundary_leak events would
   give more immediate visibility on critical boundary changes.

f) Boundary evidence references
   The boundary summary table renders evidenceReferences count inline.
   Deep-links to specific evidence package records are deferred until
   the evidence packages detail view (P12-G or later) is implemented.
