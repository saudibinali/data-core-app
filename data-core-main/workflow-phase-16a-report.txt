1. SUBSCRIPTION MODEL
================================================================================

Table: workspace_subscriptions (lib/db/src/schema/workspace-subscriptions.ts)

Fields:
  - id, workspaceId (unique — one record per tenant/workspace)
  - commercialAccountId (optional), activeContractTermId (optional)
  - subscriptionCode, subscriptionName
  - status: trial | active | grace_period | past_due | suspended | terminated | archived
  - statusReason, startDate, endDate, renewalDate
  - gracePeriodEndsAt, suspensionStartedAt, terminationDate
  - planName, internalNotes
  - createdBy, updatedBy, createdAt, updatedAt

Rules:
  - subscriptionCode unique per workspace (composite unique index)
  - No hard delete — use archived status
  - Coexists with legacy tenant_subscriptions (P13 metadata); P16-A is separate state model

================================================================================
2. STATUS LIFECYCLE (transition helper only — no enforcement)
================================================================================

Helper: artifacts/api-server/src/lib/workspace-subscription-transitions.ts

Allowed:
  trial → active | grace_period
  active → grace_period | past_due | suspended | terminated
  grace_period → active | past_due | suspended
  past_due → active | suspended
  suspended → active | terminated
  terminated → archived

Blocked:
  archived → any
  terminated → active (direct)
  suspended → trial
  past_due → trial

================================================================================
6. VALIDATION
================================================================================

  - Tenant must exist
  - commercialAccountId / activeContractTermId must belong to same tenant
  - subscriptionCode required; unique per workspace
  - startDate <= endDate; renewalDate within period (+ 90d buffer after endDate)
  - gracePeriodEndsAt after endDate when status is grace_period
  - Status change requires reason (min 10 chars)
  - Forbidden payload keys: payment, Stripe, card, tax, ledger, etc.
  - PATCH /subscription rejects inline status changes
