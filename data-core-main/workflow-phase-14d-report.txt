P14-D COMPLETION REPORT — Platform User Audit & Activity Tracking
================================================================================
Date: 2026-05-16

────────────────────────────────────────────────────────────────────────────────
SUMMARY
────────────────────────────────────────────────────────────────────────────────

P14-D adds read-only structured visibility into platform audit events stored in
the existing activity_logs table. Platform Owner and authorized roles can now
see who created platform users, who changed statuses/roles, who was blocked by
permissions, who attempted actions on protected accounts, and who modified tenant
lifecycle/subscription/entitlements — when, with what result, and why.

Implementation is:
  - Purely additive — no schema changes, no existing route modifications
  - Read-only throughout — no delete, edit, export, or SIEM hooks
  - Metadata always redacted before leaving the server
  - Permission-gated at both backend (requireAnyPlatformPermission) and
    frontend (hasAnyPlatformPermissionClient)
  - Consistent with the fixed 8-role matrix from P14-B

SECTION 1 — CURRENT AUDIT MODEL REVIEW
────────────────────────────────────────────────────────────────────────────────

Table: activity_logs (lib/db/src/schema/activity.ts)

Columns available:
  id          — serial primary key
  ticketId    — FK to tickets (nullable — null for platform events)
  userId      — FK to users (actor who performed the action; nullable on delete)
  action      — text, machine-readable event identifier
  metadata    — text, JSON string with event-specific context
  createdAt   — timestamp with timezone, defaults to now()
  workspaceId — integer FK (nullable — NULL for all platform-level events)
  busEventId  — UUID text, links to workspace_event_logs bus event

Fields NOT present as top-level columns (embedded in metadata JSON):
  targetUserId    — ID of the affected user (string in JSON)
  targetEmail     — email of the affected user
  targetName      — display name of the affected user
  result          — "success" | "blocked" | "denied" | "failed"
  reason          — human-readable reason for the action
  blockedReason   — specific reason the action was blocked
  resourceType    — type of resource affected (e.g. "workspace")
  resourceId      — ID of the resource affected

Platform scope filter:
  workspaceId IS NULL  →  platform-level events (P14-A/B/C write with null workspaceId)

Actor join:
  LEFT JOIN users ON activity_logs.userId = users.id
  → exposes actorEmail (users.email) and actorDisplayName (users.fullName)

Audit write pattern in platform-users.ts:
  writeAuditLog(actorId, action, metadata) →
    db.insert(activityLogsTable).values({ userId: actorId, action,
      metadata: JSON.stringify(metadata), workspaceId: null })

Known platform events written by P14-A/B/C:
  platform_user_created
  platform_user_create_blocked
  platform_user_status_changed
  platform_user_status_change_blocked
  platform_user_role_changed
  platform_user_role_change_blocked
  platform_permission_denied        (written by requirePlatformPermission middleware)
  protected_root_action_blocked
  platform_user_access_policy_violation

Known tenant events (tenants.ts):
  tenant_lifecycle_changed          (workspace lifecycle transitions)
  tenant_subscription_updated       (subscription metadata changes)
  tenant_entitlement_override_updated (entitlement batch overrides)

Limitations (intentionally left for P14-E or later):
  - No targetUserId column — filtering "events targeting user X" requires
    PostgreSQL JSONB cast at query time (implemented but performance-limited
    for very large datasets)
  - Actor info unavailable if userId was set null on user deletion (expected —
    shown as "System" in UI)
  - No dedicated audit indexes for action / workspaceId IS NULL combinations
    (existing idx_activity_created_at + idx_activity_user_id cover common queries)
  - metadata JSON has no enforced schema — unknown events rely on fallback config

SECTION 3 — METADATA REDACTION HELPER
────────────────────────────────────────────────────────────────────────────────

New file: artifacts/api-server/src/lib/redact-audit-metadata.ts

Sensitive key patterns (7 total, case-insensitive, substring-based):
  password · token · secret · apikey · authorization · cookie · credential

Matching strips _/- before comparing so "api_key", "api-key" both match "apikey".

Behavior:
  - Recursive through nested objects and arrays
  - Matching key's value replaced with "[REDACTED]"
  - Non-sensitive primitives (string, number, boolean) passed through unchanged
  - null / undefined returned as-is (no throw)
  - Original value never mutated (new objects created)

Exports:
  redactAuditMetadata(value: unknown): unknown
  parseAndRedactMetadata(json: string|null|undefined): Record<string,unknown>|null
    → parses JSON string, redacts, returns null on invalid JSON or empty input
  SENSITIVE_KEY_PATTERNS  (readonly)
  REDACTED_VALUE          ("[REDACTED]")

SECTION 4 — BACKEND READ APIs
────────────────────────────────────────────────────────────────────────────────

New file: artifacts/api-server/src/routes/platform-activity.ts

GET /platform/activity
  Auth:   requireAuth + requireAnyPlatformPermission(["platform.activity.read","audit.read"])
  Query params:
    actorId       string (optional) — filter by actor userId
    targetUserId  string (optional) — app-level filter on metadata.targetUserId
    action        string (optional) — exact match on action column
    group         string (optional) — app-level filter after enrichment
    result        string (optional) — app-level filter on metadata.result
    severity      string (optional) — app-level filter on event config severity
    from          ISO date string (optional)
    to            ISO date string (optional)
    limit         integer (default 50, max 200)
    cursor        integer (optional) — last item's id for next-page cursor

  Response:
    { items: EnrichedAuditItem[], nextCursor: number | null }

  EnrichedAuditItem fields:
    id, actorId, actorEmail, actorDisplayName
    targetUserId, targetEmail, targetDisplayName
    action, actionLabel, actionLabelAr, group, severity
    result, reason, blockedReason
    resourceType, resourceId
    metadataSafe   ← parseAndRedactMetadata() output (never raw JSON)
    createdAt      ← ISO string

  Pagination: cursor = last item id, WHERE id < cursor, ORDER BY createdAt DESC
  Scope:      WHERE workspaceId IS NULL (platform events only)
  Join:       LEFT JOIN users ON userId = users.id (actor name/email)

GET /platform/users/:userId/activity
  Auth:   same permission check
  Params: userId (positive integer — must be a platform user, workspaceId IS NULL)
  Query:  limit (default 20, max 100), cursor
  Response: same EnrichedAuditItem shape

  Scope:  WHERE workspaceId IS NULL AND
            (userId = :userId OR metadata::jsonb->>'targetUserId' = :userId::text)
  → Returns events where user is actor OR target
  → 400 for invalid userId, 404 if platform user not found
  → Includes blocked attempts (blocked events written to activity_logs by P14-A/B/C)

Registered in: artifacts/api-server/src/routes/index.ts
  → router.use(platformActivityRouter) — after platformMeRouter

Safety guarantees (backend):
  ✅ No DELETE / PUT / POST on audit logs
  ✅ No export endpoint
  ✅ No SIEM integration
  ✅ metadataSafe always goes through parseAndRedactMetadata before response
  ✅ workspaceId IS NULL — workspace events are never exposed here
  ✅ 403 for workspace users / insufficient permission
  ✅ Logging via req.log (pino) — never console.log
