Order: WORKFLOW_EFFECTIVENESS_SCORE_ORDER (unstable→highly_effective)

  Score             Tier       Order  Display colour   Label
  ──────────────────────────────────────────────────────────────────────────────
  unstable          critical   0      red              "Unstable"
  inconsistent      attention  1      amber            "Inconsistent"
  acceptable        neutral    2      slate            "Acceptable"
  effective         good       3      emerald          "Effective"
  highly_effective  excellent  4      teal             "Highly Effective"

Design principles:
  - No "failing" or "bad" labels — the lowest tier is "Unstable" (operational, not punitive).
  - Descriptions use governance process language, not disciplinary language.
  - "highly_effective" is the aspirational target — no "perfect" or "compliant" wording.
  - EffectivenessBadge renders a 2xl bold headline + description in tier colour.
  - Badge variant also available via colour map for inline use.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

8.1  Sort order (priority)

  1st: unresolvedFrequency (highest first)
  2nd: escalationFrequency (tiebreaker, highest first)

  This ensures policies with the most unresolved violations bubble to the top,
  giving platform operators immediate visibility into attention areas without
  requiring any filtering.

8.2  Attention threshold thresholds (empirical starting points)

  confirmedViolationRate ≥ 0.70  → amber highlight
    Rationale: 70%+ of violations confirmed means the policy is frequently
    being breached. Not necessarily a policy failure — may indicate a high-
    activity area requiring closer review.

  falsePositiveRate ≥ 0.40  → amber highlight
    Rationale: 40%+ false positives indicates potential policy over-sensitivity,
    noisy triggers, or unclear policy scope.

  escalationFrequency ≥ 0.50  → amber highlight
    Rationale: 50%+ of violations escalating suggests violations are consistently
    reaching review thresholds.

  NOTE: These are display thresholds only. No automated action is triggered
  when a threshold is exceeded. The operator reviews and decides.

8.3  Stability score in context of policy table

  The policyStabilityScore field uses the same WORKFLOW_EFFECTIVENESS_SCORE_MAP
  keys as the workflow-level score. The StabilityBadge component renders the
  human-readable tier label (e.g. "Acceptable") with tier colouring.

8.4  No policy edit controls

  The table has no edit buttons, no tuning controls, no auto-recommendation
  links. A read-only note below the table explains the attention threshold
  display logic.

  The escalation distribution chart currently shows a two-bucket split

  chart component.

  a comparison panel or sparkline column for trend-per-policy.

  does not currently expose a workspace filter — it shows platform-wide
  aggregates only. Future: add a workspace selector to the filter bar.
