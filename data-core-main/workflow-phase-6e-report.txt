Contract-first adherence
  All 5 governance endpoints were added to the OpenAPI spec FIRST, codegen
  was run, and the dashboard page uses ONLY the generated hooks from
  @workspace/api-client-react. No raw fetch() calls in the frontend.

Read-only invariant
  The dashboard contains zero mutation actions (no cancel, no force-timeout,
  no alert dismissal). The only POST call is POST /governance/events which
  is a log-only observability sink with no DB writes.

Pure utility layer
  governance-utils.ts has zero React/DOM imports and is entirely testable in
  a node environment. The injectable `now: Date` parameter on isCapturedAtStale
  and capturedAtAge enables deterministic testing without fake timers.

Polling model
  All 4 queries use TanStack Query's refetchInterval: 30_000. Stale detection
  at 3× the polling interval (90s) accounts for up to 2 missed poll cycles
  before surfacing a stale warning to the user.

Observability chain
  Frontend   →  POST /governance/events
  Server     →  req.log.info({ workspaceId, actorId, action, counts, severity })
  P6-D layer →  GET endpoint handlers already emit governance_api_* events
  Net result →  Dashboard load emits: governance_dashboard_loaded +
                4× governance_api_* events (one per data query)
