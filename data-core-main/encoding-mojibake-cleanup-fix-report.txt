Encoding Mojibake Cleanup Fix Report
=====================================
Date: 2026-05-18
Scope: artifacts/ops-platform, artifacts/api-server, lib/db, lib/api-client-react (source trees)

SUMMARY
-------
Removed UTF-8 mojibake and normalized UI-facing punctuation to plain ASCII across
377 source files. Added a guard test and a reusable fix script (Unicode escapes only).
No business logic, schema, or UI layout changes. P16-D not started.

ROOT CAUSE
----------
UTF-8 punctuation (smart quotes, em/en dashes, ellipsis, box-drawing) was either:
1) Mis-decoded as Windows-1252 mojibake (e.g. U+00E2 U+20AC U+2122 for apostrophe), or
2) Stored as Unicode punctuation (U+2014, U+2026) where ASCII was preferred for UI.

PATTERNS FOUND AND REPLACED
---------------------------
Mojibake (UTF-8 read as Windows-1252):
  U+00E2 U+20AC U+2122  ->  '     (right single quote / apostrophe)
  U+00E2 U+20AC U+02DC  ->  '     (left single quote)
  U+00E2 U+20AC U+0153  ->  "     (left double quote)
  U+00E2 U+20AC U+009D  ->  "     (right double quote)
  U+00E2 U+20AC U+00A6  ->  ...   (ellipsis)
  U+00E2 U+20AC U+201C  ->  -     (en dash mojibake)
  U+00E2 U+20AC U+201D  ->  -     (em dash mojibake)
  U+00E2 U+201D U+20AC  ->  -     (box-drawing mojibake)
  U+00C2 U+00B7         ->  " - " (middle dot mojibake)
  U+00C2 U+00A0         ->  space (nbsp mojibake)
  U+FFFD                ->  (removed replacement char)

Unicode punctuation normalized to ASCII (UI + comments):
  U+2014 em dash        ->  -
  U+2013 en dash        ->  -
  U+2018/U+2019 quotes  ->  ' / "
  U+201C/U+201D quotes  ->  " / "
  U+2026 ellipsis       ->  ...
