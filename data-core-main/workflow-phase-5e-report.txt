1. OBJECTIVE
────────────
Replace the lightweight POST /workflows/:id/activate toggle with a full
immutable publish pipeline. Every activation now:
  (a) Runs the full governance validation (all 4 engine passes + per-step checks).
  (b) Writes an immutable, append-only version row to `workflow_definition_versions`.
  (c) Closes the previously active version row (sets deactivated_at = NOW()).
  (d) Updates the definition atomically: status=active, version++, currentVersionId,
      publishedAt, publishedBy.

The version history table is append-only. Version rows are never deleted or
overwritten — only `deactivated_at` and `deactivated_by` are set when a version
is superseded or the workflow is deprecated.


2. DATABASE SCHEMA (lib/db/src/schema/workflows.ts)
────────────────────────────────────────────────────
New table: `workflow_definition_versions`
  - id                 SERIAL PRIMARY KEY
  - definitionId       INTEGER NOT NULL → workflow_definitions(id) ON DELETE RESTRICT
  - workspaceId        INTEGER NOT NULL → workspaces(id) ON DELETE CASCADE
  - version            INTEGER NOT NULL
  - steps              JSONB NOT NULL             ← frozen at publish time
  - conditions         JSONB NOT NULL             ← frozen at publish time
  - triggerEvent       TEXT NOT NULL              ← frozen at publish time
  - name               TEXT NOT NULL              ← frozen at publish time
  - nameAr             TEXT
  - changeNotes        TEXT                       ← admin publish notes
  - publishedBy        INTEGER
  - publishedAt        TIMESTAMP NOT NULL DEFAULT NOW()
  - deactivatedAt      TIMESTAMP                  ← NULL = currently active
  - deactivatedBy      INTEGER
  - validationSummary  JSONB                      ← full validation result

  Unique index: (definitionId, version)   — prevents duplicate version numbers
  Index:        (workspaceId, definitionId, deactivatedAt)

New columns on `workflow_definitions`:
  - version          INTEGER NOT NULL DEFAULT 0  ← monotonic publish counter
  - currentVersionId INTEGER                     ← FK to versions (plain int, no circular FK)
  - publishedAt      TIMESTAMP
  - publishedBy      INTEGER

Design note: `ON DELETE RESTRICT` on the definitionId FK prevents hard-deleting
a workflow that has a version history. Soft-delete (status=archived, deletedAt set)
remains the only safe removal path.

4. ROUTE HANDLER (artifacts/api-server/src/routes/workflows.ts)
────────────────────────────────────────────────────────────────
POST /workflows/:id/activate — full publish transaction:
  1. Load definition (workspaceId scoped).
  2. If status === 'active': return 200 { alreadyActive: true } (idempotent).
  3. If status === 'archived': return 404.
  4. Run validateWorkflow(steps, triggerEvent).
  5. If errors.length > 0: emit `workflow_publish_blocked` audit log, return 422.
  6. DB transaction:
     a. If currentVersionId exists → close previous version row
        (SET deactivatedAt = NOW(), deactivatedBy = req.userId)
     b. INSERT workflow_definition_versions with full frozen snapshot +
        validationSummary JSONB + changeNotes.
     c. UPDATE workflow_definitions:
        status = 'active', isActive = true, version = oldVersion + 1,
        currentVersionId = newVersionId, publishedAt = NOW(), publishedBy = userId
  7. Emit `workflow_publish_started` + `workflow_publish_completed` audit log events.
  8. Return 200 WorkflowPublishResult with version metadata.

PATCH /workflows/:id — deprecation path updated:
  - When status transitions to 'deprecated': wraps in a transaction that also
    closes the active version row (sets deactivatedAt, deactivatedBy) and clears
    currentVersionId = null on the definition.
  - Emits `workflow_deprecated` audit log event.

GET /workflows/:id/versions — new endpoint:
  - Returns all version rows for the definition, ordered by version DESC.
  - Joins publishedBy → users table for publishedByName display field.
  - Returns 404 if definition not found (workspaceId scoped).


5. ENGINE UPDATE (artifacts/api-server/src/lib/workflows/engine.ts)
────────────────────────────────────────────────────────────────────
workflowVersion at execution INSERT:
  Before P5-E: always null (commented "populated in P7-A").
  After  P5-E: (def as unknown as { version?: number | null }).version ?? null
  - def.version = 0 → legacy definition never re-published (pre-P5-E).
  - def.version = N → version N was active at trigger time.
  - NULL → pre-schema execution (DB column didn't exist yet).

The observability log at execution creation also updated to record the live
def.version value.

8. INVARIANTS & CONTRACTS
──────────────────────────
I1  Version numbers are monotonically increasing per workflow.
    Implementation: newVersion = currentVersion + 1 in the publish transaction.
    Enforcement: UNIQUE(definitionId, version) DB constraint.

I2  Version rows are append-only.
    Implementation: no UPDATE on frozen fields (steps, conditions, triggerEvent,
    name, version, publishedBy, publishedAt). Only deactivatedAt/By are ever set.
    Enforcement: application-layer only (no DB trigger needed at this scale).

I3  status='active' ↔ exactly one open version row (deactivatedAt IS NULL).
    Enforcement: publish transaction closes previous row before writing new one.

I4  status='deprecated' ↔ zero open version rows.
    Enforcement: PATCH deprecation path closes the active version row atomically.

I5  workflowVersion on executions = def.version at trigger time.
    Purpose: allows historical audit queries to JOIN workflow_definition_versions
    on (definitionId, version) to recover the exact published config used.

I6  Publish is blocked if validation returns any errors.
    The `valid` flag from validateWorkflow() is the gate. Warnings and notices
    are recorded in validationSummary but do not block.
