3. NEW AUDIT EVENTS (structured log fields)
────────────────────────────────────────────────────────────────────

  approval_requested              — step fired, notifications sent, execution pausing
  approval_requested_no_approvers — step fired but no approvers resolved (skip)
  approval_decision_recorded      — approved; guarded UPDATE won; loop resuming
  approval_decision_rejected      — rejected; guarded UPDATE won; execution failed
  approval_resume_blocked_ttl_expired  — TTL expired, approve blocked
  approval_rejection_blocked_ttl_expired — TTL expired, reject blocked
  approval_replay_prevented       — duplicate decision attempt detected (pre-flight)
  transition_race_lost            — guarded UPDATE returned empty (concurrent operation)

All events carry: executionId, workflowId, workspaceId, workflowVersion,
approvalId (where applicable), decidedBy, approvalStepIndex.

5. EXACT-ONCE DECISION GUARANTEE (two-layer model)
────────────────────────────────────────────────────────────────────

Layer 1 — Pre-flight (route handler):
  • SELECT workflow_approvals WHERE executionId + stepIndex
  • If found → 409 APPROVAL_ALREADY_DECIDED + approval_replay_prevented event
  • Catches: client double-submit, browser retry, UI bug

Layer 2 — Atomic gate (executor):
  • UPDATE WHERE status='waiting_approval' (resume also adds cancelRequested=false)
  • .returning() — empty result = race lost → TRANSITION_RACE_LOST
  • Catches: concurrent approve+approve, approve+timeout, approve+reject

Both layers log structured events with full traceability fields.
Approval records are only inserted AFTER the guarded UPDATE wins.
No orphaned records can exist.
