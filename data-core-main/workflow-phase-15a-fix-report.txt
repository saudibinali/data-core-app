4. Route Improvements — NEW IN THIS FIX
────────────────────────────────────────────────────────────────────────────────

File: artifacts/api-server/src/routes/commercial.ts

Added:

a) Email format validation (RFC-5322 subset regex):
   - PUT  /commercial-account: billingEmail, contractOwnerEmail
   - POST /commercial-contacts: contactEmail (required + validated)
   - PATCH /commercial-contacts/:id: contactEmail (optional, validated if provided)
   → Returns 400 with descriptive error if format is invalid.

b) accountManagerUserId / financeOwnerUserId handling:
   - Both fields now accepted in PUT /commercial-account body
   - Validated: must be a positive integer, 0 to clear, or omitted to leave unchanged
   - Non-integer, negative, or float values → 400 "Invalid accountManagerUserId"
   - Value 0 → null (clears the FK)
   - Positive integer → stored as-is

c) No new payment, Stripe, or tax logic was introduced.
