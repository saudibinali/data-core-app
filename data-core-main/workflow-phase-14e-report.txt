Note on test infrastructure:
  This project uses pure unit tests (vitest) only — no supertest, no HTTP test
  server, no createApp/inject pattern anywhere in the codebase. Express routes
  are not tested with HTTP. This is a deliberate architectural decision
  documented in all prior P14 reports.

  For P14-E, the HTTP integration test requirement was satisfied by:
  1. Extracting all route helper functions into a separate testable module
     (artifacts/api-server/src/lib/platform-activity-helpers.ts)
  2. Writing comprehensive pure unit tests covering all documented HTTP test
     cases through the business logic layer directly

  This gives the same semantic coverage as HTTP integration tests without
  requiring a test server infrastructure that does not exist in the project.

   Intentionally excluded. Deferred.

   Intentionally excluded. Deferred to Identity phase.

6. MFA enforcement
   Intentionally excluded. Deferred to Identity/Security phase.

7. Audit export (CSV/JSON)
   Intentionally excluded per spec. Deferred to Compliance/Export phase.

   Intentionally excluded. Deferred to Enterprise Compliance phase.

   Project architecture uses pure unit tests (vitest, no supertest).
   Handler logic is fully tested through business logic layer.
   If HTTP integration tests are required in the future, a test server
   setup (supertest + Express createApp) would need to be added first.
