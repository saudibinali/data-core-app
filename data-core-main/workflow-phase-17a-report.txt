  - platform_user_type (platform_owner | platform_admin | platform_operator)

  - platform_reactivated_by, platform_reactivated_at, platform_reactivation_reason

Existing fields retained:
  - email (normalized lowercase on create), fullName as displayName
  - status: invited | active | disabled | suspended | locked
  - platformRoleCode, isRootOwner, isProtected, lastLoginAt
  - No password on this model (auth separate)
  - No hard delete

Unique index: users_email_lower_unique_idx on lower(trim(email))

================================================================================
2) LIFECYCLE RULES
================================================================================
Create (POST /platform/users):
  - email required + valid, normalized lowercase
  - displayName required (min 2 chars)
  - userType or roleCode required; platform_owner create blocked
  - default status: invited
  - platformCreatedBy set from actor

Profile update (PATCH /platform/users/:userId):
  - displayName, jobTitle, department, phone only
  - email update blocked (EMAIL_UPDATE_NOT_SUPPORTED)
  - isRootOwner update blocked (ROOT_OWNER_FLAG_IMMUTABLE)
  - protected/root targets require policy (non-root cannot manage root)

Disable / suspend / lock (PATCH .../status):
  - reason required (min 10) + confirmation
  - platform.users.disable OR legacy platform.users.status.update
  - records platformDisabledBy/At/Reason

Reactivate (status → active):
  - platform.users.reactivate OR legacy platform.users.status.update
  - records platformReactivatedBy/At/Reason

Guards:
  - LAST_ACTIVE_OWNER_PROTECTED — cannot disable last active platform owner
  - PROTECTED_ACCOUNT — non-root cannot disable root/protected
  - No DELETE route
